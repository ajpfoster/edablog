Posted by Ash Foster on May 13th, 2019

Problem solving katas

A simple problem that confused me this sprint was object access for the children in the manipulating objects kata. To solve it I had to trial and error dot and bracket notation for object access. This is when you use "." or [ ] to access properties stored on an object. In this example the object was Terah and her children property was set using dot notation. I was able to read error messages and console logs from the tests to help and I learned that practising what I'm covering in documentation is critical to a better understanding of the material.

This was also true when I was able to understand the arrow functions I had seen in some of the test functions and apply its use to local variables for inbuilt functions. This was a combination of course material and googling/following up study on free code camp but I was also able to quickly prototype similar solutions to the examples using loops but the ability of arrow functions to elegantly get the same result in one line made me glad I was able to understand what was happening with the example code.
